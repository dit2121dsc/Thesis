Load balancing is essential for datacenterTerms networks. However, prior solutions have significant limitations: they either are oblivious to congestion or involve a daunting and time-consuming parametertunning task over their heuristics for achieving good performance. Thus, we ask: is it possible to learn to balance datacenterTerms traffic? While deep reinforcement learningCSO (DRLTerms) sounds like a good answer, we observe that it is too heavyweight due to the long decisionmaking latency. Therefore, we introduce BULBTerms, a lightweight and automated datacenterTerms load balancer. BULBTerms learns link weights to guide the end-hosts to spread traffic, so as to free the central agent from quick flow-level decision-making. BULBTerms offline trains a DRLTerms agent for optimizing link weights but employs an imitation learningCSO based approach to faithfully translate this agent's DNNTerms to a decision tree for online deployment. We implement a BULBTerms prototype with a popular machine learningCSO framework and evaluate it extensively in ns-3. The results show that BULBTerms achieves up to 36.6%/56.4%, 19.9%/42.5%, 35.9%/54.8%, and 45.1%/67.7% better average/tail flow completion time than ECMP, CONGA, LetFlow, and Hermes, respectively. Moreover, BULBTerms reduces the decision latency by 175 times while incurring only 2% performance loss after converting the DNNTerms into a decision tree.
