Intrusion DetectionCSO System (IDSTerms) / SoftwareCSO Defined Networking (SDN) cooperative firewallCSO systems attract much attention recently because they have many advantages such as dynamic network configuration with SDN and scalable IDSCSO hosts. In IDSTerms/SDN cooperative firewallCSO systems, an SDN switch relays traffic between external and internal networks and mirrors the traffic to an IDSTerms host. The IDSTerms host monitors mirrored traffic and tells the SDN controller to block malicious traffic according to attack detectionCSO. Therefore, when the IDSTerms hosts are overloaded, it causes a large delay between attack detectionCSO by the IDSCSO host and malicious traffic blocking in the SDN switch, which increases the amount of malicious traffic reaching the server. In this paper, we propose a configuration to reduce the loads on the IDSTerms hosts by excluding benign flows from packet monitoring. By using the proposed configuration, IDSCSO hosts reduce the long delay in blocking because the proposed configuration mitigates the performance degradation in IDSCSO hosts due to monitoring of all traffic, including known benign flows. We implemented the proposed configuration on our IDSTerms/SDN cooperative firewallCSO systems using real devices. Then we evaluated the effectiveness of delay reduction with and without excluding benign flows when varying the number of IDSTerms hosts and the notification method used for blocking malicious traffic, namely using Syslog messages and using RESTTerms APITerms. As a result, we confirm that the proposed configuration reduces the delay time in all cases investigated. Furthermore, we confirmed that Syslog notifications are faster than RESTTerms APITerms notifications in all cases.
